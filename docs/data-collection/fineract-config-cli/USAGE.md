## Installation

### Prerequisites

```bash
# Check Java version (17+ required)
java -version

# Should output: openjdk version "17.0.x" or higher
```

### Option 1: Download Pre-built JAR

```bash
# Download latest release
wget https://github.com/apache/fineract/releases/download/v1.0.0/fineract-config-cli.jar

# Make executable (optional)
chmod +x fineract-config-cli.jar

# Test installation
java -jar fineract-config-cli.jar --version
```

### Option 2: Build from Source

```bash
# Navigate to project directory
cd fineract/docs/data-collection/fineract-config-cli

# Build with Maven
mvn clean package -DskipTests

# JAR location
ls -lh target/fineract-config-cli*.jar

# Create alias for convenience
alias fineract-cli='java -jar $(pwd)/target/fineract-config-cli.jar'
```

### Option 3: Docker

```bash
# Pull latest image
docker pull ghcr.io/apache/fineract/fineract-config-cli:latest

# Create alias
alias fineract-cli='docker run --rm -v $(pwd):/workspace ghcr.io/apache/fineract/fineract-config-cli:latest'

# Test
fineract-cli --version
```

## Basic Commands

### 1. Import Configuration

**Minimal import:**
```bash
java -jar fineract-config-cli.jar \
  --fineract.url=http://localhost:8080/fineract-provider \
  --fineract.tenant=default \
  --fineract.user=mifos \
  --fineract.password=password \
  --import.files=config.yml
```

**With environment variables:**
```bash
export FINERACT_URL=http://localhost:8080/fineract-provider
export FINERACT_TENANT=default
export FINERACT_USER=mifos
export FINERACT_PASSWORD=password

java -jar fineract-config-cli.jar --import.files=config.yml
```

**Multiple files (glob pattern):**
```bash
# Import all YAML files in config directory
java -jar fineract-config-cli.jar --import.files='config/*.yml'

# Import specific phases
java -jar fineract-config-cli.jar --import.files='config/phase{1,2,3}.yml'
```

**Before import:**

# Validate YAML syntax and structure

## Configuration

### Application Properties File

Create `application.yml` in the same directory as the JAR:

```yaml
# application.yml
fineract:
  url: http://localhost:8080/fineract-provider
  tenant: default
  user: mifos
  password: ${env:FINERACT_PASSWORD}  # Read from environment
  ssl-verify: true
  connection-timeout: 30000
  read-timeout: 60000

  auth:
    type: basic  # or oauth2

    # OAuth2 configuration (if auth.type=oauth2)
    oauth2:
      token-url: http://keycloak:9000/realms/fineract/protocol/openid-connect/token
      client-id: fineract-client
      client-secret: ${env:OAUTH_CLIENT_SECRET}
      grant-type: client_credentials

import:
  files:
    locations: /config/*.yml
  validate: true
  parallel: false
  force: false
  dry-run: false

  remote-state:
    enabled: true
    cache-duration: 3600

  managed:
    office: full
    role: full
    user: no-delete
    staff: full
    client: no-delete
    loanProduct: full
    savingsProduct: full

  var-substitution:
    enabled: true
    prefix: "${"
    suffix: "}"

logging:
  level:
    root: INFO
    org.apache.fineract.config: DEBUG
    org.apache.fineract.config.service: TRACE
  file:
    name: logs/fineract-config-cli.log
    max-size: 10MB
    max-history: 30
```

### Environment Variables

```bash
# Create .env file
cat > .env << 'EOF'
# Fineract Connection
FINERACT_URL=http://localhost:8080/fineract-provider
FINERACT_TENANT=default
FINERACT_USER=mifos
FINERACT_PASSWORD=password

# Authentication

# OAuth2 (if using)

FINERACT_AUTH_OAUTH2_CLIENT_ID=fineract-client
FINERACT_AUTH_OAUTH2_CLIENT_SECRET=secret123

# Import Settings
IMPORT_FILES_LOCATIONS=/config/*.yml
IMPORT_VALIDATE=true
IMPORT_PARALLEL=false
IMPORT_FORCE=false
IMPORT_DRY_RUN=false

# State Management
IMPORT_REMOTE_STATE_ENABLED=true

# Managed Resources
IMPORT_MANAGED_OFFICE=full
IMPORT_MANAGED_CLIENT=no-delete
IMPORT_MANAGED_LOANPRODUCT=full

# Variable Substitution
IMPORT_VAR_SUBSTITUTION_ENABLED=true

# Logging
LOGGING_LEVEL_ROOT=INFO
LOGGING_LEVEL_ORG_APACHE_FINERACT_CONFIG=DEBUG
EOF

# Load environment
source .env

# Run
java -jar fineract-config-cli.jar
```

### Command-Line Options (Complete List)

```bash
# Fineract Connection
--fineract.url=<url>                    # Base URL
--fineract.tenant=<tenant>              # Tenant identifier
--fineract.user=<username>              # Username

--fineract.ssl-verify=<true|false>      # SSL verification
--fineract.connection-timeout=<ms>      # Connection timeout
--fineract.read-timeout=<ms>            # Read timeout

# Authentication
--fineract.auth.type=<basic|oauth2>     # Auth method
--fineract.auth.oauth2.token-url=<url>  # OAuth2 token URL
--fineract.auth.oauth2.client-id=<id>   # OAuth2 client ID
--fineract.auth.oauth2.client-secret=<secret>  # OAuth2 secret

# Import
--import.files=<path>                   # Config file path(s)
--import.validate=<true|false>          # Validate before import
--import.parallel=<true|false>          # Parallel processing
--import.force=<true|false>             # Force import (skip checksum)
--dry-run                               # Preview mode

# State Management
--import.remote-state.enabled=<true|false>      # Remote state tracking
--import.remote-state.cache-duration=<seconds>  # Cache duration

# Managed Resources
--import.managed.<entity>=<full|no-delete>      # Resource management mode

# Variable Substitution
--import.var-substitution.enabled=<true|false>  # Enable variable substitution

# Logging
--logging.level.root=<LEVEL>                    # Root log level
--logging.level.org.apache.fineract.config=<LEVEL>  # CLI log level
--logging.file.name=<path>                      # Log file path

# Utility
--help                                  # Show help
--version                               # Show version
```

## Common Workflows

### Workflow 1: Initial System Setup

**Step 1: Prepare configuration**
```bash
# Create system config
cat > phase1-system.yml << 'EOF'
tenant: default

systemConfig:
  currency:
    code: USD
    name: US Dollar
    decimalPlaces: 2

  workingDays:
    recurrence: FREQ=WEEKLY;BYDAY=MO,TU,WE,TH,FR
    repaymentReschedulingType: MOVE_TO_NEXT_WORKING_DAY
    extendTermForDailyRepayments: false

  codes:
    - name: CustomerType
      values:
        - name: Individual
          position: 1
          isActive: true
        - name: Business
          position: 2
          isActive: true
EOF
```

**Step 2: Validate**
```bash
java -jar fineract-config-cli.jar validate --file=phase1-system.yml
```

**Step 3: Dry-run**
```bash
java -jar fineract-config-cli.jar --dry-run --import.files=phase1-system.yml
```

**Step 4: Apply**
```bash
java -jar fineract-config-cli.jar --import.files=phase1-system.yml
```

**Step 5: Verify**
```bash
# Export and compare
java -jar fineract-config-cli.jar export --output=verify.yml --phases=1
diff phase1-system.yml verify.yml
```

### Workflow 4: Migration Between Environments

**Export from source:**
```bash
# Export production config (excluding operational data)
java -jar fineract-config-cli.jar export \
  --fineract.url=https://prod.example.com/fineract-provider \
  --fineract.user=admin \
  --fineract.password=${PROD_PASSWORD} \
  --output=prod-export.yml \
  --phases=1,2,3,4  # System config, security, accounting, products
```

**Sanitize for target:**
```bash
# Replace environment-specific values
sed -i 's/prod.example.com/staging.example.com/g' prod-export.yml

# Replace passwords (all exported as ***EXPORTED***)
sed -i 's/\*\*\*EXPORTED\*\*\*/StagingPass123!/g' prod-export.yml
```

**Import to target:**
```bash
# Dry-run first
java -jar fineract-config-cli.jar \
  --fineract.url=https://staging.example.com/fineract-provider \
  --fineract.user=admin \
  --fineract.password=${STAGING_PASSWORD} \
  --dry-run \
  --import.files=prod-export.yml

# Apply
java -jar fineract-config-cli.jar \
  --fineract.url=https://staging.example.com/fineract-provider \
  --fineract.user=admin \
  --fineract.password=${STAGING_PASSWORD} \
  --import.files=prod-export.yml
```

### Workflow 5: Rollback Changes

**Before making changes:**
```bash
# Always export before major changes
java -jar fineract-config-cli.jar export \
  --output=backup-$(date +%Y%m%d-%H%M%S).yml \
  --phases=all
```

**If changes need to be reverted:**
```bash
# Import previous backup
java -jar fineract-config-cli.jar \
  --import.files=backup-20251120-100000.yml

# Note: This will restore configuration but won't delete
# entities created after backup. For full rollback:
java -jar fineract-config-cli.jar \
  --import.managed.office=full \
  --import.managed.staff=full \
  --import.managed.client=full \
  --import.files=backup-20251120-100000.yml
```

## Advanced Usage

### OAuth2 Authentication

**Configure Keycloak authentication:**
```yaml
# application.yml
fineract:
  url: http://localhost:8080/fineract-provider
  tenant: default

  auth:
    type: oauth2
    oauth2:
      token-url: http://keycloak:9000/realms/fineract/protocol/openid-connect/token
      client-id: fineract-client
      client-secret: ${env:OAUTH_CLIENT_SECRET}
      grant-type: client_credentials
      scope: openid profile email
```

**Run with OAuth2:**
```bash
export OAUTH_CLIENT_SECRET=your-secret
java -jar fineract-config-cli.jar --import.files=config.yml
```

### Variable Substitution

**Environment variables:**
```yaml
systemConfig:
  globalConfig:

      value: ${env:AWS_ACCESS_KEY}
      enabled: true

    - name: amazon-S3_secret_key

      enabled: true
```

**File contents:**
```yaml
systemConfig:
  globalConfig:

      value: ${file:/run/secrets/smtp_password}
      enabled: true

    - name: database_password
      value: ${file:/etc/fineract/db_password}
      enabled: true
```

**System properties:**
```yaml
systemConfig:
  globalConfig:
    - name: api_base_url
      value: ${sys:fineract.url}
      enabled: true
```

**Recursive substitution:**
```yaml
systemConfig:
  globalConfig:
    - name: secret_location
      value: /run/secrets
      enabled: true

    - name: db_password
      value: ${file:${env:SECRET_LOCATION}/db_password}
      enabled: true
```

### Managed Resource Modes

**Full management (delete unmanaged):**
```bash
# Offices: delete offices not in YAML
java -jar fineract-config-cli.jar \
  --import.managed.office=full \
  --import.files=config.yml
```

**No-delete mode (preserve unmanaged):**
```bash
# Clients: preserve clients not in YAML
java -jar fineract-config-cli.jar \
  --import.managed.client=no-delete \
  --import.files=config.yml
```

**Mixed modes:**
```bash
# Different modes per entity type
java -jar fineract-config-cli.jar \
  --import.managed.office=full \
  --import.managed.staff=full \
  --import.managed.client=no-delete \
  --import.managed.loanProduct=full \
  --import.managed.loanAccount=no-delete \
  --import.files=config.yml
```

### Parallel Processing

**Enable for large imports:**
```bash
# Process entities in parallel (faster but higher load)
java -jar fineract-config-cli.jar \
  --import.parallel=true \
  --import.files=large-config.yml
```

**Configure thread pool:**
```yaml
# application.yml
import:
  parallel: true
  thread-pool-size: 10  # Default: CPU cores * 2
```

### Force Import (Skip Checksum)

**Ignore checksums:**
```bash
# Reapply configuration even if unchanged
java -jar fineract-config-cli.jar \
  --import.force=true \
  --import.files=config.yml
```

**Use case:** Configuration drift detected, force realignment

## Troubleshooting

### Connection Issues

**Problem:** `Connection refused`
```
ERROR: Failed to connect to Fineract: Connection refused (Connection refused)
```

**Solutions:**
1. Check Fineract is running: `curl http://localhost:8080/fineract-provider/actuator/health`
2. Verify URL in configuration
3. Check firewall/network settings
4. Ensure correct port (8080 vs 8443)

**Problem:** `401 Unauthorized`
```
ERROR: Authentication failed: 401 Unauthorized
```

**Solutions:**
1. Verify credentials: `--fineract.user` and `--fineract.password`
2. Check tenant exists: `--fineract.tenant`
3. For OAuth2, verify token URL and client credentials
4. Check user has required permissions

### Validation Errors

**Problem:** Invalid YAML syntax
```
ERROR: Validation failed: Invalid YAML syntax at line 42
Expected ':' but found '-'
```

**Solution:**
1. Check YAML indentation (use spaces, not tabs)
2. Validate with online YAML validator
3. Use IDE with YAML support (VS Code, IntelliJ)

**Problem:** Missing required field
```
ERROR: Validation failed: Missing required field 'currencyCode' in loanProduct 'Personal Loan'
```

**Solution:**
1. Review entity requirements in CONFIGURATION.md
2. Add missing field to YAML
3. Check for typos in field names

### Import Failures

**Problem:** Entity not found (dependency)
```
ERROR: Failed to load staff 'Jane Advisor': Office 'Branch Office' not found
```

**Solution:**
1. Ensure dependencies exist (office before staff)
2. Check entity names match exactly (case-sensitive)
3. Use correct resolution syntax: `$office.Branch Office`

**Problem:** Checksum mismatch
```
WARN: Configuration checksum unchanged, skipping import. Use --force to override.
```

**Solution:**
1. This is intentional (idempotency)
2. Use `--force` if you need to reapply: `--import.force=true`

### Performance Issues

**Problem:** Slow imports
```
INFO: Import completed in 5 minutes 43 seconds
```

**Solutions:**
1. Enable parallel processing: `--import.parallel=true`
2. Increase thread pool: Set `import.thread-pool-size=20`
3. Optimize network (use local Fineract instance for testing)
4. Split into smaller batches

### 3. Pre-Import Validation

```bash
#!/bin/bash
# validate-and-import.sh

CONFIG_FILE=$1

echo "Validating configuration..."
java -jar fineract-config-cli.jar validate --file=${CONFIG_FILE}

if [ $? -eq 0 ]; then
  echo "Validation passed. Proceeding with dry-run..."
  java -jar fineract-config-cli.jar --dry-run --import.files=${CONFIG_FILE}

  echo "Review changes above. Proceed? (yes/no)"
  read CONFIRM

  if [ "$CONFIRM" == "yes" ]; then
    echo "Importing configuration..."
    java -jar fineract-config-cli.jar --import.files=${CONFIG_FILE}
  else
    echo "Import cancelled."
  fi
else
  echo "Validation failed. Please fix errors and try again."
  exit 1
fi
```

### 4. Automated Backups

```bash
#!/bin/bash
# backup.sh - Run before major changes

BACKUP_DIR=backups
TIMESTAMP=$(date +%Y%m%d-%H%M%S)
BACKUP_FILE="${BACKUP_DIR}/backup-${TIMESTAMP}.yml"

mkdir -p ${BACKUP_DIR}

echo "Exporting current configuration..."
java -jar fineract-config-cli.jar export \
  --output=${BACKUP_FILE} \
  --phases=all

echo "Backup saved to: ${BACKUP_FILE}"

# Keep only last 30 backups
ls -t ${BACKUP_DIR}/backup-*.yml | tail -n +31 | xargs rm -f
```

### 6. Security Best Practices

**Never commit passwords:**
```yaml
# ❌ BAD - Hardcoded password
users:
  - username: admin
    password: SecretPass123!

# ✅ GOOD - Environment variable
users:
  - username: admin
    password: ${env:ADMIN_PASSWORD}
```

**Use secrets management:**
```bash
# Store in secrets manager (e.g., HashiCorp Vault)
export ADMIN_PASSWORD=$(vault kv get -field=password secret/fineract/admin)

# Or use file-based secrets (Kubernetes)
export ADMIN_PASSWORD=$(cat /run/secrets/admin_password)
```
