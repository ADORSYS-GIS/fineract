# Export Functionality Guide

## Overview
The Export functionality allows you to extract Fineract configuration data to YAML format for backup, migration, or documentation purposes.

## Usage

### Basic Export (All Phases)
```bash
export --output=exported-config.yml
```

### Export Specific Phases
```bash
# Single phase
export --output=phase2.yml --phases=2

# Multiple phases
export --output=config.yml --phases=1,2,3

# All phases explicitly
export --output=full-config.yml --phases=all
```

### Advanced Export with Options (Java API)
```java
// Export only active entities
ExportOptions options = ExportOptions.builder()
    .activeOnly(true)
    .build();
exportService.exportConfiguration("output.yml", options);

// Export configuration only (no transactions)
ExportOptions options = ExportOptions.configurationOnly();
exportService.exportConfiguration("config-only.yml", options);

// Export specific phase with filtering
ExportOptions options = ExportOptions.builder()
    .phases(Set.of(1, 2))
    .excludeEntityTypes(Set.of("dataTables"))
    .includeComments(true)
    .build();
exportService.exportConfiguration("filtered.yml", options);
```

## Security & Data Protection

### Password Sanitization
**CRITICAL**: All user passwords are replaced with `***EXPORTED***` placeholder.

⚠️ **WARNING**: Exported configurations with this placeholder **CANNOT** be imported directly. You must:
1. Replace `***EXPORTED***` with secure passwords
2. Use strong passwords (8+ chars, mixed case, numbers, symbols)
3. Ensure users change passwords on first login

### Sensitive Data Exclusion
By default, the export excludes:
- System users (`system`)
- System-defined codes
- Disabled roles
- Inactive entities (when `activeOnly=true`)

## Best Practices

### 1. Regular Backups
```bash
# Daily config backup
export --output=backup-$(date +%Y%m%d).yml --phases=1,2,3,4
```

### 2. Pre-Migration Export
```bash
# Export full configuration before major changes
export --output=pre-migration-$(date +%Y%m%d-%H%M%S).yml --phases=all
```

### 3. Documentation Generation
```bash
# Export for documentation purposes
export --output=system-documentation.yml --phases=1,2
```

### 4. Partial Export for Testing
```bash
# Export only products for testing
export --output=products-only.yml --phases=4
```

### 5. Sanitized Export for Sharing
```bash
# Export without sensitive data (default behavior)
export --output=shareable-config.yml
# Passwords already sanitized
# No user credentials exposed
```

## Troubleshooting

### Export Fails with 401 Unauthorized
- Check Fineract API credentials in `application.yml`
- Verify user has `READ` permissions for all exported entities

### Missing Entities in Export
- Verify entities exist in Fineract (check via UI/API)
- Check entity filtering options (`excludeEntityTypes`)
- Review phase selection (`--phases` parameter)

### Empty Export File
- Verify Fineract instance has data
- Check API connectivity
- Review logs for API errors

### Password Import Fails
- Export contains `***EXPORTED***` placeholder
- Must replace with actual passwords before import
- Use strong password policy
